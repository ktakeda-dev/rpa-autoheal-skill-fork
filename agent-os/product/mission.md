# Product Mission

## Pitch

**Browser Automation Framework (Claude AI x Playwright)** は、ビジネスユーザーから開発者まで、誰もがブラウザ自動化を構築・実行・改善できるAI駆動の自動化フレームワークです。

**「作って、試して、直す」を全部AIがやる**

自然言語の指示からYAMLワークフローを自動生成し、決定論的な高速実行を実現。失敗時にはAIがその場でリカバリし、原因を特定してYAMLを自動修正。同じ失敗は二度と起きません。

## The Problem

### ブラウザ自動化の「三重苦」

従来のブラウザ自動化ツールは、以下の問題を同時に解決できていません。

1. **作成の難しさ**: プログラミング知識が必要で、ビジネスユーザーが自分で作れない
2. **実行の不安定さ**: AIエージェントは柔軟だが遅い。従来RPAは速いが壊れやすい
3. **改善の手間**: 失敗時の原因特定と修正が手動で、同じ問題が繰り返し発生

**Our Solution:** ハイブリッドアプローチで三重苦を同時に解決

- **作成**: 自然言語 → AI がYAML自動生成（誰でも作れる）
- **実行**: YAML → 最適化コード生成 → 高速決定論的実行 + 失敗時のみAIリカバリ
- **改善**: AIが失敗原因を特定 → レポート生成 → YAML自動修正（学習する自動化）

## Differentiators

### 1. 決定論的高速実行 + AIリカバリのハイブリッド

| ツール | 作成 | 速度 | リカバリ | 自己改善 |
|--------|------|------|----------|----------|
| Power Automate Desktop | AI支援 | 高速 | 停止 | 手動修正 |
| AIエージェント (Browser-Use等) | AI | 低速 | AIリトライ | なし |
| **本フレームワーク** | **AI** | **高速** | **AIリカバリ** | **YAML自動修正** |

従来のAIエージェントとは異なり、通常実行時はAI推論なしで高速に動作。失敗時のみPlaywright MCPを通じてAIが介入し、画面状態を理解してリカバリを実行します。

### 2. 失敗から学習する自動化

単なるリトライではなく、AIが失敗の根本原因を分析。YAMLワークフローを自動修正するため、同じ失敗が二度と発生しません。これにより運用コストが継続的に低下します。

### 3. 組織の知識資産として蓄積

- **Skill (SKILL.md)**: Claudeへのドメイン知識として永続化
- **YAML**: 人間が読める形式でワークフローを管理
- **Git連携**: バージョン管理により変更履歴を追跡

自動化の知見が個人ではなく組織に蓄積され、誰でも活用・改善できます。

### 4. Claude Codeサブスクリプション内で完結

追加のSaaS契約不要。Claude Codeのサブスクリプション内で全機能が利用可能。コスト効率に優れ、導入障壁が低い。

## Key Features

### Core Features (コア機能)

- **自然言語からYAML生成**: 日本語/英語の指示からワークフローYAMLを自動生成
- **高速決定論的実行**: YAMLからPlaywrightコードを生成し、AI推論なしで高速実行
- **AIリカバリ**: 失敗時にPlaywright MCPでスナップショット取得、状態理解、タスク完遂

### Intelligence Features (インテリジェンス機能)

- **失敗分析レポート**: 失敗原因の自動特定とレポート生成
- **YAML自動修正**: 分析結果に基づくワークフローの自動改善
- **Skill管理**: ドメイン知識の蓄積と活用

### Enterprise Features (エンタープライズ機能)

- **バージョン管理連携**: Git によるワークフロー履歴管理
- **実行ログ・監査**: 実行履歴の記録と追跡
- **拡張性**: Skills/MCPsによる機能拡張

## Future Vision

### Phase 1: ブラウザ自動化の完成
現在のコア機能を安定化し、ブラウザ自動化における最高の体験を提供

### Phase 2: 選択的AI実行
ステップごとにRPA実行とAI実行を選択可能に。確実性が必要な箇所はRPA、柔軟性が必要な箇所はAI。

### Phase 3: 並列タスク実行
複数タスクの並列実行により、大規模自動化のスループットを向上

### Phase 4: ブラウザを超えて
デスクトップアプリ操作、ファイル操作など、ブラウザ以外の自動化領域への拡張
